04

AIOps

// 監視・ログスタックを活用し IT 運用を AI で知能化

予測型アラート

predict_linear() でディスク・メモリ枯渇を事前検知。 CPU スパイク・Pod 再起動率のトレンドアラートも実装。 AltertManager の inhibit_rules でノイズを抑制。

ログ異常検知

ADTK (Python) による統計的異常検知を CronJob で定期実行。 検知結果を Prometheus Pushgateway 経由で Grafana ダッシュボードに可視化。

LLM アラートサマリ

Claude API

AlertManager からアラートを受信し Claude API (claude-haiku) で人間可読なサマリを生成。 結果を Grafana アノテーションとして可視化し、障害対応の初動を高速化。

自動修復 (Runbook)

Argo Events でアラートをトリガーに Argo Workflows を起動。 Pod 再起動・スケールアウト等の修復アクションを自動実行する Runbook を整備。

05

MCP Server (proxmox-lab-mcp)

Raspberry Pi 5 上で稼働するホームラボ専用 MCP サーバー。 Claude Code から HTTP/SSE で接続し、自然言語でインフラ操作・調査が可能。

Claude Code (Windows)
    │  HTTP/SSE
    ▼
Raspberry Pi 5 ← MCP Server
    ├── proxmoxer  → Proxmox API
    ├── terraform CLI
    ├── ansible CLI
    ├── kubectl
    └── ArgoCD REST API

[ Proxmox ]

ノード/VM 状態確認・起動停止・スナップショット・クラスター健全性

[ Terraform / Ansible ]

plan・apply・playbook 実行。破壊的操作は confirm 必須の安全設計。

[ kubectl / Helm ]

Pod ログ・イベント確認・apply・ロールアウト管理・Helm アップグレード

[ ArgoCD ]

全アプリの sync/health 確認・sync 実行・hard refresh

[ Lab Utility ]

Wake-on-LAN・SSH 実行・ポート確認・クラスター全体健全性サマリー

[ 登録コマンド ]

claude mcp add --transport sse proxmox-lab http://<pi-ip>:8000/sse
